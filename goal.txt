xcono/asr — standalone ASR/STT module for the realtime voice agent.

This is the "ears" of a voice pipeline that was decoupled out of xcono/voices
into three Go modules. This file is the canonical record of that decoupling: the
architecture, the resolved decisions, and the remaining work.

The service includes:
- silero-go VAD
- ASR/STT model interface (GigaAM local, ElevenLabs remote)
- an importable, NATS-free facade (Stream/Batch/IsSpeaking)
- cmd/vox: a standalone debug binary (embedded NATS lives here ONLY)

Config: config.json — VAD timing, STT provider, NATS (cmd/vox only).
Dev-ops: avoid Python where possible; GigaAM/Qwen3-ASR run as Python servers
(no Go equivalent), pointed at via base_url.
Development: built iteratively by AI coding agents.

# Decoupling

1. Facts
  - The voice pipeline is split into three Go modules, consumed in-process:
      xcono/asr    — STT "ears": VAD + transcribe (cgo/ONNX). THIS repo.
      xcono/tts    — TTS "mouth": synthesis client (pure-Go).
      xcono/voices — LLM brain (act, bus, llm) + the audio device.
  - voices imports asr + tts as library deps; streaming flows over Go channels /
    emit callbacks. NO NATS on the library path (NATS can't stream bytes).
  - Python ASR servers moved here: llm/giga-asr -> asr/llm/giga3,
    llm/qwen3-asr -> asr/llm/qwen3. TTS server moved to tts/llm/qwen3-tts.
  - The old ./stt copy (module github.com/xcono/stt) was superseded by this repo
    (github.com/xcono/asr) and deleted; this repo carries its full history.

2. Resolved decisions
  - 2 module repos (asr, tts) + voices. Rationale: per-component focus for
    AI-driven dev, independent versioning, and asr's heavy cgo/ONNX/PortAudio
    deps stay out of the pure-Go tts client.
  - In-process Go modules, NOT network services. NATS dropped from the library
    path; it survives only behind cmd/vox (the standalone debug binary).
  - voices knows ONE rule about STT: when VAD becomes TRUE, interrupt the TTS
    that is currently speaking (event-driven via SpeechStart, or IsSpeaking()).
  - Audio contracts (fixed): STT/VAD = 16 kHz/16-bit/LE/mono; TTS = 24 kHz/mono/int16 LE.
  - asr facade: Events() (Stream), Transcribe() (Batch), IsSpeaking() (atomic bool).
  - tts facade: Stream (SynthesizeStream), Batch (Synthesize), Status (ready/processing/speaking).
  - Shared pkg/audio is COPIED into each repo (no shared module yet).
  - go.work at the xcono/ parent (use ./asr ./tts ./voices) for local dev; voices
    also has local `replace` directives so plain `go build` works.
